# Documentation Rules

## Цель документации

Документация проекта нужна для сохранения структуры знаний и быстрого восстановления контекста работы.

Документы не должны дублировать друг друга.

---

# Правила обновления файлов

## MISSION.md

Обновляется только при изменении:

- главной цели;
- направления развития;
- принципов обучения.

Не изменяется при обычном продвижении по модулям.

---

## ROADMAP.md

Обновляется после завершения значимых этапов обучения.

Используется для отслеживания общего прогресса.

Пример:

- изучен новый технологический блок;
- завершен модуль;
- добавлен новый этап.

---

## STATUS.md

Главный рабочий документ.

Обновляется после каждого учебного занятия.

Содержит:

- текущий модуль;
- выполненные задачи;
- текущую задачу;
- следующий шаг.

---

## STACK.md

Обновляется при освоении новых технологий.

Содержит:

- освоенные инструменты;
- изучаемые инструменты;
- планируемые технологии.

---

## mentor.md

Обновляется только при изменении правил взаимодействия с наставником.

---

# Общие правила

1. Не создавать документы без конкретной функции.
2. Не дублировать информацию между файлами.
3. Документация должна помогать разработке, а не создавать дополнительную работу.
4. Каждый документ должен отвечать на один конкретный вопрос.
